# DESIGN

Verve is an outdoor activity frontend. Design changes should support fast scanning, calm activity review, and clear form workflows.

## Principles
- Keep primary actions obvious and reachable.
- Preserve the existing Verve palette from `src/assets/main.css`.
- Prefer recognizable icons for common actions.
- Avoid explanatory UI copy that describes implementation details or keyboard shortcuts.
- Do not nest UI cards inside other cards.
- Keep fixed-format UI stable with explicit dimensions or responsive constraints.

## Accessibility
- Interactive elements need accessible names.
- Use semantic controls before custom controls.
- Maintain visible focus states.
- Avoid text overlap on mobile and desktop.
